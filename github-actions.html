🚀 Getting Started with GitHub Actions

In today’s DevOps-driven world, automation is not optional—it’s essential. One of the most powerful and widely used tools for CI/CD is GitHub Actions. Whether you're deploying applications, running tests, or provisioning infrastructure, GitHub Actions helps you automate everything directly from your repository.

🧠 What is GitHub Actions?

GitHub Actions is a CI/CD (Continuous Integration and Continuous Deployment) platform built into GitHub. It allows you to automate workflows such as:

  • Building code
  • Running tests
  • Deploying applications
  • Managing infrastructure (Terraform, Kubernetes, etc.)

All of this is defined using simple YAML files inside your repository.

📁 Key Concepts

🔹 Workflow

A workflow is an automated process defined in a YAML file.

🔹 Event

Triggers that start a workflow (e.g., push, pull request).

🔹 Job

A set of steps that run on the same machine.

🔹 Step

Individual tasks (run commands, use actions).

🔹 Runner

The machine (VM) where your workflow runs.

🚀 Your First GitHub Actions Workflow

Create a file:

.github/workflows/main.yml

📄 Example: Basic CI Pipeline

name: CI Pipeline

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run a simple command
        run: echo "Hello, DevOps 🚀"

⚙️ Real-World Example: Terraform Workflow

name: Terraform CI/CD

on:
  push:
    branches: [ "main" ]

jobs:
  terraform:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3

      - name: Terraform Init
        run: terraform init

      - name: Terraform Validate
        run: terraform validate

      - name: Terraform Plan
        run: terraform plan

🔐 Managing Secrets

Never hardcode credentials. Use GitHub Secrets:

  1. Go to Repo → Settings → Secrets
  2. Add secrets like:
  • GCP_CREDENTIALS
  • AWS_ACCESS_KEY

Use them in workflow:

env:
  GOOGLE_CREDENTIALS: ${{ secrets.GCP_CREDENTIALS }}

🚀 Advanced Use Cases

🔹 Deploy to Cloud (GCP example)

- name: Authenticate to GCP
  uses: google-github-actions/auth@v2
  with:
    credentials_json: ${{ secrets.GCP_CREDENTIALS }}

🔹 Multi-Environment Deployment

on:
  push:
    branches:
      - dev
      - stage
      - main

👉 You can map:

  • dev → Dev environment
  • stage → Staging
  • main → Production

🔹 Matrix Builds (Parallel Jobs)

strategy:
  matrix:
    node-version: [14, 16, 18]

🧠 Best Practices

  • Keep workflows small and modular
  • Use reusable actions
  • Store secrets securely
  • Separate CI and CD pipelines
  • Use environment-based deployments
  • Add approvals for production

⚠️ Common Mistakes

  • Hardcoding credentials
  • Running everything in one job
  • Not using caching (slower builds)
  • Ignoring logs and debugging

🔥 Pro Tips

  • Use self-hosted runners for custom workloads
  • Add caching for faster builds
  • Use environments + approvals for production safety
  • Integrate with Terraform + Kubernetes + ArgoCD

🎯 Final Thoughts

GitHub Actions is more than just CI/CD—it’s a full automation engine for your DevOps lifecycle.

  • Automate infrastructure
  • Deploy applications seamlessly
  • Maintain consistent environments across dev, stage, and prod

Automation is not about doing things faster — it's about doing them reliably every time.
