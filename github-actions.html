Getting Started with GitHub Actions

Automate your DevOps workflow using GitHub Actions, from builds and tests to Terraform and Kubernetes deployments. This page is your quick-start guide for creating practical pipeline automation inside GitHub.

Why GitHub Actions matters

In today’s DevOps-driven world, automation is not optional—it’s essential. GitHub Actions lets you automate everything directly from your repository, including deployments, tests, and infrastructure provisioning.

What is GitHub Actions?

GitHub Actions is a CI/CD platform built into GitHub that enables workflows for:

  • Building code
  • Running tests
  • Deploying applications
  • Managing infrastructure (Terraform, Kubernetes, etc.)

Key concepts

  • Workflow — an automated process defined in a YAML file.
  • Event — triggers that start a workflow, like push or pull_request.
  • Job — a set of steps that run on the same machine.
  • Step — an individual task, such as running a command or using an action.
  • Runner — the machine on which the workflow executes.

Your first workflow

Create .github/workflows/main.yml to start automating your DevOps pipeline.

name: CI Pipeline

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run a simple command
        run: echo "Hello, DevOps 🚀"

Terraform workflow

Build a practical pipeline for infrastructure validation and planning.

name: Terraform CI/CD

on:
  push:
    branches: [ "main" ]

jobs:
  terraform:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Terraform
        uses: hashicorp/setup-terraform@v2

      - name: Terraform Init
        run: terraform init

      - name: Terraform Plan
        run: terraform plan
